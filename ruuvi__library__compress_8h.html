ret_type_t rl_compress ( const rl_data_t *const  data,
uint8_t *const  block,
const size_t  block_size,
rl_compress_state_t *const  state 
)

Ruuvi Library compress function. Takes a sensor data sample in and appends it to given data block.

It is assumed that data is appended in linear order, new sample has always greater timestamp than previous.

Parameters
[in]dataSensor data to compress, 1 sample.
[out]blockPointer to buffer to which compressed data is placed.
[in]block_sizeSize of block.
[in,out]stateIn: State of compression algorithm before adding latest data. Out: State of compression algorithm after adding latest data.
Return values
RL_SUCCESSIf compression was successful.
RL_ERROR_NULLIf given a NULL as a parameter.
RL_COMPRESS_ENDIf the block is considered full. Block is ready to be stored to non-volatile memory.
RL_COMPRESS_OUT2BIGIf compressed data is larger than uncompressed data. In this case uncompressed data should be stored, compressed data may be invalid if it doesn't fit into block.

uint8_t *  block,

rl_compress_state_t *  state,

Ruuvi Library decompress function. Looks up next sample after given timestamp and returns it via output parameter.

Usage:

ret_type_t status;
rl_data_t data;
uint8_t* block = get_next_block();
size_t block_size = DATA_BLOCK_SIZE;
rl_decompress_state_t state = { 0 }; // zeroed state initially.
timestamp_t start_timestamp = 0; // Get all the data from boot.
while(RL_COMPRESS_SUCCESS == status)
{
// This pseudocode fails if no data is found and it misses last sample.
status = rl_decompress(&data, block, block_size, &state, &start_timestamp);
do_something_with_data(&data);
}
Parameters
[out]dataNext sample from block. Not modified if no data was found in block.
[in]blockPointer to compressed buffer with sensor data.
[in]block_sizeSize of block.
[in,out]stateIn: State of decompression algorithm before decompressing next data point. Out: State of decompression algorithm after decompressing next data point.
[in,out]start_timestampIn: Earliest timestamp to accept. Out: Timestamp of returned data.
Return values
Statusof decompression, such as more available or not_found.
